Utah Transit Authority
Operations Analytics, Project Engagement
Oct 2024 → Jan 2025

Failure-sequence diagnostics + vendor SLA accountability across a 250-machine fleet.

The core work centered on ticketing operations: reconstructing rider sessions, isolating the error sequences that caused failed purchases, and comparing those failures against what the vendor actually surfaced to Control Center. SQL, Python, Power BI, and Power Query supported the analysis across the newly installed 250+ TVM fleet.

Key Accomplishments
  • Built dashboards tracking 20+ KPIs across 7 lines, cutting report-gen time from 8 hrs/wk to 15 min automated.
  • Sessionized machine logs and mapped the repeated user flows most likely to end in a failed purchase.
  • Ranked failure classes by frequency, retry behavior, and recoverable leakage across the fleet.
  • Diagnosed the TVM vendor was filtering non-critical error classes at the source, leaving Control Center with an incomplete view of machine outages.
  • Authored the scope accountability document used in vendor negotiations — line-by-line contracted scope vs. field error data; vendor agreed to fix pipeline and accept SLA enforcement.
Highlighted Story • Vendor Accountability
Turned a vendor-support failure into enforceable SLAs using data.

Situation. 250+ new TVMs across TRAX, FrontRunner, and bus; riders were encountering failed purchase sessions, but the vendor wasn't surfacing the full set of error alerts to Control Center as contracted.

Action. Three tracks in parallel: rebuilt rider sessions from raw machine logs to isolate the action sequences behind failed purchases, cross-referenced vendor logs against what Control Center actually received, and used field testing plus a scope-accountability document to map contractual support scope line by line against observed failures.

Result. Vendor agreed to fix the error-reporting pipeline and accepted ongoing SLA enforcement. Detection moved from rider-complaint-driven to near-real-time. The scope accountability doc became the standing reference for future vendor-support reviews.
